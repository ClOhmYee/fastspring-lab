# Spring Boot 기반 서버 구성

## 동기

1. Java는 다양한 알고리즘 문제를 풀면서 익숙해졌지만, Spring은 개념만 가볍게 숙지하였으며 실제로 구현한 경험은 없음

2. 과제를 수행하며 단일 책임 원칙(SRP) 등 객체 지향 프로그래밍(OOP)을 최대한 고려한 프로젝트를 진행하는 것이 중요하다고 생각함

3. 이전에 진행하고자 계획했지만 시간 상 진행하지 못 한 개인 프로젝트의 축소판으로 계획하여 이어 도전하고자 함

## 기능

1. 기본적인 서버를 구축

2. FastAPI 서버에 데이터를 전송하고 추후 응답을 받는 구조

3. FastAPI 서버에 데이터를 보낼 수 있는 간단한 인터페이스 구성

## 이슈

1. 기능 기반 패키지 구조를 구상했었으나, 레이어 기반 패키지 구조를 통해 역할을 명확하게 드러낼 수 있을 것이라 판단해 구조를 변경함

2. RestTemplate를 활용하여 코드를 구성하던 중, 해당 클래스에 대한 지원 중단을 확인해 WebClient, RestClient 등 다른 대체제를 확인함. 따라서 이전에 구성한 RestTemplate 대신 RestClient로 대체해 코드를 구성하기로 결정함

3. 이전 과제는 각 기능이 독립적이었으므로 Unit Test 구현이 쉬웠지만, 해당 프로젝트에서는 Controller, Service, Client, Dto로 서로 의존하여 작동한다는 점에서 테스트 방법에 대한 이해가 필요함
